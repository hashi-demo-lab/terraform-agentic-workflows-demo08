USE CASE 1 — CONSUMER MODULE UPLIFT

Automated Module Version Upgrades for HCP Terraform Consumers

When a module author publishes a new version to the private registry, consumers need their workspace code updated. This workflow combines Dependabot detection with AI-powered analysis to classify risk, adapt code, and decide whether to auto-merge, request review, or block.

Key Principles

Never Auto-Merge Majors

Major version bumps always require human review, even with zero breaking changes detected. The semantic version signal itself warrants attention.

DESTROY Escalates

Any resource destruction in the plan escalates to at minimum NEEDS-REVIEW. Infrastructure deletion demands human oversight.

Conservative Bias

When uncertain, the system recommends needs-review over auto-merge. False positives are preferable to missed breaking changes.

Security Trumps Semver

Security findings of HIGH or above always escalate regardless of version type. IAM, encryption, and exposure changes get special scrutiny.
